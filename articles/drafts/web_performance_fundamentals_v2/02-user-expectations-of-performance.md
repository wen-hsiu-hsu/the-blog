# 網頁效能與使用者體驗：關鍵數字與研究根據

講師提出網頁效能重要的三大理由：**使用者體驗**、**搜尋引擎優化（SEO）** 與**線上廣告**。本篇聚焦於使用者體驗這條主線。

## 使用者體驗的本質

使用者體驗指的是網站能否符合甚至超越使用者的預期。當使用者造訪一個網站，他們對載入速度已有既定期望，這個期望來自過去使用類似網站的經驗、當下要完成的任務，以及這個網站在他們日常生活中的重要程度。

講師特別指出一點：我們無法替每位使用者升級網路或設備，所以只能想辦法讓網站本身變快。

## 人機互動研究告訴我們的數字

以下資料來自講師引用的學術研究，而非廠商自行委託的調查：

**IEEE 研究**指出，使用者發出請求後，需要在 **2 秒以內**感受到某種回應，才算有效溝通。超過 2 秒，使用者就會感到中斷，注意力開始漂移。

**可用性工程研究**進一步細分了幾個關鍵門檻：

| 回應時間     | 使用者感受                                           |
| ------------ | ---------------------------------------------------- |
| 小於 0.01 秒 | 感覺即時，完全沒有察覺延遲                           |
| 約 1 秒      | 注意到延遲，但不至於打斷操作流程                     |
| 10 秒以上    | 幾乎所有使用者都會感到挫折，注意力與操作流程就此中斷 |

## 數據：慢速網站的代價

講師引用一本網頁效能領域書籍中的大型研究數據：

- **40%** 的使用者會在等待超過 **3 秒**後離開網站
- **75%** 曾感受到網站緩慢的使用者，之後**不會再回來**

這代表效能問題不只是體驗不佳，而是直接影響使用者留存與信任。

> 後兩項（SEO 與線上廣告）將於後續篇章分別說明。

## 複習

### 根據可用性工程研究，在什麼樣的回應時間下，使用者會感覺操作幾乎是即時完成的？

小於 0.01 秒

### 如果網站載入時間超過三秒，有多少比例的使用者會離開？

40% 的使用者

### 根據 IEEE 的回應時間研究，要達到有效溝通，回應需要在多快時間內送達？

在請求發出後的兩秒以內

### 有多少比例的使用者不會再回到他們認為緩慢的網站？

75% 的使用者

### 在什麼樣的回應時間下，使用者幾乎都會感到挫折並中斷操作流程？

10 秒以上
